SECTION 2 — VISUAL DNA LAW

These are not guidelines. These are laws. Nothing ships that violates them.

COLORS:
Primary background: #0A0A0A — near black, not pure black, has depth
Secondary background: #1A1A1A — for cards and elevated surfaces
Tertiary background: #2A2A2A — for borders and subtle depth
Primary accent: #00D9FF — electric cyan. Used for: highlights, active states, key data points, technical elements, futuristic feel
Secondary accent: #C9A84C — gold. Used for: premium elements, KingCam identity, royalty signals, CTAs, crown moments
Text primary: #FFFFFF — pure white, bold
Text secondary: #E8E8E3 — warm off-white for body copy
Text accent: #00D9FF or #C9A84C depending on context
NEVER use: gray backgrounds, white backgrounds, light mode anything, generic blue, generic green, stock photo color palettes

TYPOGRAPHY:
Headlines: Playfair Display — serif, elegant, commands attention, royalty energy
Subheads and UI labels: Inter — clean, modern, readable
Body copy: Inter — never smaller than 14px on mobile
All caps used for: section labels, status badges, platform names, agent names
Never use: Comic Sans, Arial, Times New Roman, any font that feels corporate or generic

HIERARCHY ON EVERY VISUAL:
1. ONE dominant element — the thing your eye goes to first
2. ONE supporting element — context or subhead
3. ONE action element — button or CTA
That is the maximum. Three elements. Never more on a single visual surface.

SPACING LAW:
Minimum 40px padding on all sides
Elements never touch each other — minimum 16px between any two elements
Text never touches an image edge
White space is not empty space — it is luxury

VISUAL BENCHMARKS — every output must feel like it belongs next to these:
Apple product launches — clean, intentional, every pixel justified
Dior campaign visuals — luxury, desire, editorial quality
NBA playoff graphics — bold, high energy, sports premium
A$AP Rocky creative direction — culture, taste, swagger without trying
None of these are cheap. None of these are generic. That is the standard.

SECTION 4 — TELEGRAM OUTPUT STANDARD

Telegram is the primary revenue channel. Every message must be worth reading.

SUBSCRIBER CHANNEL MESSAGES:
Maximum 4 sentences
One clear CTA at the end
No placeholder text ever
No markdown asterisks visible to the reader
No walls of explanation
Visual if included must pass the three-element layout law
Frequency: maximum 3 per day, minimum 4 hours between messages
No posts between 11pm and 8am Dallas time

APPROVED MESSAGE FORMATS:

FORMAT A — CONTENT DROP (when delivering locked content):
Line 1: One intriguing hook about what is inside. Make them need to know.
Line 2: The specific value or transformation inside.
Line 3: The access instruction.
[Button: Unlock — $X]

EXAMPLE OF WHAT THIS LOOKS LIKE:
"The moment a creator stops trading time for content and starts trading content for income — everything changes.
Tonight's drop shows exactly how three creators on CreatorVault crossed $1,000/month in their first 30 days.
Tap below to get inside."
[Button: Unlock Full Access — $29]

FORMAT B — VALUE DROP (free value that builds trust):
Line 1: A specific insight that immediately sounds valuable.
Line 2: Why it matters for them specifically.
Line 3: Optional link or "more inside" reference.

EXAMPLE:
"Most creators leave 60% of their income on the table because they only post — they never follow up.
On CreatorVault, the follow-up is automated. You post once. The system closes for you.
That is the difference."

FORMAT C — OFFER MESSAGE (selling something):
Line 1: The outcome they are buying, not the product.
Line 2-3: Two or three specific things included.
Line 4: Price and what makes this moment specific.
[Button: Get it — $X]

EXAMPLE:
"Three creators got their full subscription system built and live this week.
They each now have a paid tier, a Telegram funnel, and an automated follow-up sequence running.
Founder setup — $197 this week only."
[Button: Lock in your spot]

FORMAT D — ENGAGEMENT (building community):
One question. That is it. Make it specific enough that people actually have an answer.

EXAMPLE:
"What is the one thing stopping you from charging for your content right now?"

WHAT NEVER GOES IN A SUBSCRIBER CHANNEL:
Raw GPT output with asterisks and headers
Internal agent reports
Placeholder text with brackets
Messages about platform technical issues
Anything that requires the reader to think hard to understand what they are supposed to do

SECTION 7 — PLATFORM UI QUALITY STANDARD

Every page, dashboard, and component visible to users must meet this standard.

LAYOUT LAW:
No page that looks like a form — if information is needed, collect it progressively in a conversational flow
No walls of bullet points — use cards with visual hierarchy
No tables unless the data is genuinely tabular and comparative
No light mode anything — dark is the identity
Every page has one primary action — the user always knows what to do next

DASHBOARD QUALITY:
Numbers that matter are large and prominent — not buried in a table
Charts use platform colors — #00D9FF for trends, #C9A84C for milestones, white for neutral data
Empty states are never just "no data found" — they are invitations: "Your first subscriber is one post away. Here is how."
Loading states are never spinners on white backgrounds — they are dark screens with a KingCam brand animation

MOBILE FIRST LAW:
Every UI designed for a 375px screen first then expanded for desktop
Touch targets minimum 44px
No hover-only interactions — everything works with a tap
Text minimum 16px on mobile — never smaller

SECTION 8 — THE QUALITY GATE

Before anything posts publicly or reaches a subscriber it must pass this gate.

AUTOMATED CHECKS (implemented in code):
- No placeholder text: scan for [Name], [Amount], [$X], [LINK], TODO, placeholder
- No raw markdown: scan for **, *, ##, *** in user-facing messages
- Minimum length: 30 characters for any message
- Maximum length: 500 characters for broadcast Telegram messages
- Must contain at least one action element (link, button, or instruction)
- No duplicate messages to same recipient within 48 hours

VISUAL CHECKS (implemented as pre-post review):
- Three element maximum rule
- 40px padding minimum verified
- No overlapping text detected
- Background contrast ratio minimum 4.5:1 for readability
- Image minimum resolution 1080x1080

BRAND VOICE CHECKS (GPT-4o review before posting):
Before any subscriber-facing message posts, run it through this GPT-4o prompt:
"Review this Telegram message for a premium creator platform called CreatorVault owned by KingCam. Score it 1-10 on: directness (does it get to the point?), value (does it give the reader something?), tone (does it sound like a confident peer not a corporation?), clarity (is the CTA obvious?). If any score is below 7 — rewrite the message to meet the standard. Return only the final approved message."

Only post the GPT-4o approved version.
